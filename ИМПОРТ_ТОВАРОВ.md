# 📥 Функционал импорта товаров

## Описание

Реализован полный функционал для массового импорта товаров из CSV файлов в систему "Торнус Склад". Функция позволяет быстро заполнить базу данных товарами, загрузив файл вместо ручного добавления каждого товара.

## Структура реализации

### 1. Backend (FastAPI)

#### Функция `bulk_import_products` в `app/crud.py`

```python
def bulk_import_products(db: Session, products_data: list[dict],
                         on_duplicate: str = "skip") -> dict
```

**Параметры:**
- `db`: Сессия SQLAlchemy
- `products_data`: Список словарей с данными товаров
- `on_duplicate`: Действие при существовании товара по SKU
  - `"skip"` — пропустить (не создавать)
  - `"update"` — обновить существующий товар
  - `"error"` — ошибка (прерватить импорт)

**Возвращает:**
```json
{
  "created": 5,
  "updated": 2,
  "skipped": 0,
  "errors": [
    {"row": 10, "sku": "INVALID", "error": "SKU не может быть пусто"}
  ]
}
```

**Особенности:**
- Валидация обязательного поля `sku`
- Обработка дубликатов по SKU
- Откат транзакции при ошибках (rollback)
- Логирование ошибок с номерами строк

#### API Endpoint в `app/routers/products.py`

```
POST /products/import/csv
```

**Параметры:**
- `file` (FormData) — CSV файл
- `on_duplicate` (query param) — "skip" | "update" | "error"

**Ожидаемый формат CSV:**
```csv
name,sku,category,unit,price,description,min_stock,current_stock
Картофель,POTATO-001,Овощи,кг,50.00,Картофель сорт Беллароза,100,250
Вода (0.5л),WATER-500,Напитки,шт,30.00,,50,100
Молоко (1л),MILK-1000,Молочные,л,85.50,Молоко коровье 3.6%,30,45
```

**Требуемые колонки:**
- `name` — название товара
- `sku` — артикул (уникальный)

**Опциональные колонки:**
- `category` — категория товара
- `unit` — единица измерения (по умолчанию: "шт")
- `price` — цена в рублях (по умолчанию: 0)
- `description` — описание товара
- `min_stock` — минимальный остаток (по умолчанию: 0)
- `current_stock` — текущий остаток (по умолчанию: 0)

Добавлена универсальная функция `_post_file()` для загрузки файлов через multipart/form-data.

## Использование

### Для пользователя

1. **Подготовить CSV файл:**
   - Скачать шаблон со страницы импорта
   - Заполнить данные товаров
   - Убедиться в кодировке UTF-8

2. **Загрузить файл:**
   - Перейти на страницу "📥 Импорт"
   - Выбрать файл
   - Выбрать действие при дубликатах (skip/update/error)
   - Нажать "Импортировать"

3. **Проверить результаты:**
   - Просмотреть статистику (создано, обновлено, пропущено)
   - При наличии ошибок — исправить CSV и повторить импорт

### Для разработчика

**Интеграция с вашим приложением:**

```python
from app.crud import bulk_import_products

# Импорт товаров из списка словарей
products = [
    {"name": "Товар 1", "sku": "SKU-001", "price": 100.00, "current_stock": 50},
    {"name": "Товар 2", "sku": "SKU-002", "price": 200.00, "current_stock": 30},
]

result = bulk_import_products(db, products, on_duplicate="skip")
print(result)  # {"created": 2, "updated": 0, "skipped": 0, "errors": []}
```

Помидоры,TOMATO-001,кг,80.00,150

**Результат:**
```json

  "updated": 0,
  "skipped": 0,
  "errors": []

```

### Пример 2: Импорт с обновлением

**Параметр:** `on_duplicate=update`

Если товар с SKU уже существует, его данные будут обновлены с информацией из файла (name, price, min_stock и т.д.).

### Пример 3: Обработка ошибок

**CSV с ошибками:**
```csv
name,sku,price
Товар без SKU,,100.00
Товар 2,SKU-002,abc.00
```

**Результат:**
```json
{
  "created": 0,
  "updated": 0,
  "skipped": 0,
  "errors": [
    {"row": 1, "sku": "", "error": "SKU не может быть пусто"},
    {"row": 2, "sku": "SKU-002", "error": "could not convert string to float: 'abc.00'"}
  ]
}
```

## Технические детали

### Форматы данных

**Числовые значения:**
- Используйте точку (.) как разделитель: `150.50`
- Не используйте запятые: ❌ `150,50`

**Текстовые значения:**
- Без кавычек (CSV парсер обработает автоматически)
- UTF-8 кодировка обязательна

**Пустые значения:**
- Допустимы пустые ячейки для опциональных полей
- Будут заполнены значениями по умолчанию

### Обработка дубликатов

| Действие | Поведение |
|----------|-----------|
| `skip` | Пропустить товар, оставить существующий неизменным |
| `update` | Обновить все поля существующего товара |
| `error` | Вернуть ошибку, остановить импорт |

### Валидация данных

Проверяются:
- ✅ Обязательное поле `sku`
- ✅ Уникальность `sku` в базе (при on_duplicate="error")
- ✅ Формат числовых полей (price, min_stock, current_stock)
- ✅ Длина строковых полей

## Ограничения и особенности

- **Максимальный размер файла:** зависит от лимита FastAPI (по умолчанию ~100MB)
- **Максимальное количество товаров:** не ограничено
- **Откат при ошибках:** При ошибке в строке она пропускается, импорт продолжается (кроме on_duplicate="error")
- **Кодировка:** Только UTF-8
- **Разделитель CSV:** Запятая (`,`)
